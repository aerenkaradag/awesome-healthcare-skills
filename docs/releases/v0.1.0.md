# Release notes: v0.1.0

## Summary

First open-source release of Awesome Healthcare Skills, a collection of open-source AI agent skills for safer and more practical healthcare AI workflows. This release provides five bounded, review-oriented skills, supporting documentation, and synthetic example data with a validation script. The scope is intentionally small and community-first.

These skills support research, prototyping, data engineering, validation support, and workflow assistance. They are not medical devices, clinical decision systems, or substitutes for professional judgment.

## Included skills

- `fhir-to-ml-dataset-builder`: convert FHIR or FHIR-derived resources into structured, ML-ready tables with documented mappings.
- `ehr-care-gap-analyzer`: inspect EHR-derived tables for possible care gap signals and missing follow-up documentation, framed as review signals.
- `medical-ai-validation-report-writer`: turn model metrics, cohorts, evaluation notes, and limitations into a structured validation report.
- `medical-imaging-dataset-auditor`: audit imaging datasets for split integrity, duplicates, metadata leakage, class balance, and documentation gaps.
- `health-ai-evidence-citation-checker`: review healthcare AI answers for unsupported claims, citation quality, uncertainty language, and escalation boundaries.

Recommended first: `fhir-to-ml-dataset-builder`, `medical-ai-validation-report-writer`, and `ehr-care-gap-analyzer`.

## Example data

Synthetic, deterministic fixtures under `examples/` for testing the priority skills:

- `synthetic-fhir-bundle.json`: a compact FHIR R4 Bundle with synthetic patients and diabetes-relevant resources.
- `diabetes-cohort.csv`: a synthetic diabetes cohort with labs, dates, comorbidity flags, medication flags, and follow-up dates.
- `validation-results.csv`: synthetic model evaluation rows with subgroup fields.
- `scripts/generate_examples.py`: regenerates all files deterministically using the standard library only.
- `scripts/validate_examples.py`: validates structure, required columns, ISO dates, PHI safety, and row counts.

All example data is synthetic. It is not real patient data and is not clinical evidence.

## Safety boundaries

- These skills do not provide medical advice and do not diagnose, treat, monitor, or manage patients.
- They are not clinical decision systems and are not a substitute for professional judgment.
- Use de-identified or synthetic data unless you have proper authorization to process identifiable health data.
- Do not place PHI, patient images, direct identifiers, or sensitive notes in issues, examples, commits, or pull requests.
- Outputs require review by qualified humans before clinical, operational, legal, or regulatory use.
- The skills are not claimed to be clinically validated.

See `SECURITY.md` and the `Safety boundaries` section of `README.md` for details.

## Known limitations

- Skills are agent-readable instruction documents, not packaged libraries or hosted services.
- FHIR and EHR data vary across vendors, versions, and institutions; local code mappings may be required.
- Example data is small and synthetic, intended for testing and demonstration, not for benchmarking.
- The repository provides no clinical validation, regulatory clearance, or deployment-readiness claims.
- Continuous integration checks structure and links only; it does not assess clinical correctness.
